Each approach changes how weather-aware scheduling is surfaced to the user.

A — Buckets (Do Now / This Week / Coming Up)

Tasks grouped into rolling time buckets. Weather drives which bucket each task lands in. Clean and easy to scan — you immediately see what needs attention today.

B — Calendar dates (actual days)

AI assigns a specific recommended date to each task based on the weather forecast. Tasks beyond the forecast window get approximate labels ("Late June"). Feels more concrete and actionable.

C — Priority groups with dates on each task

Groups by urgency (Urgent / High / Routine) rather than time, but each task shows its specific recommended date. Combines priority-first scanning with precise scheduling.
